vue父组件异步请求获取数据传递给子组件

发表于 2019-07-16 | 分类于 study

　　父组件传值给子组件很简单，但是如果父组件的值是异步请求获取到的数据，然后直接通过props传递给子组件，子组件里面打印是拿不到值的。通过试验发现作以下处理是可以的。
　　1.父组件在传值前判断值是否已经存在，即异步请求是否已加载完毕，如果没有则不加载子组件。然后在子组件里面就可以直接打印props定义的值，当然也可以在data里面另外定义一个变量接收。

　　2.如果没有在父组件中处理请求还未完成就直接传值的情况，那么直接在子组件中拿props里面的值是拿不到的，因为父组件中请求还未完成，子组件就已经加载完成。这时候在子组件中可以定义一个变量，然后通过watch来监听props中值的变化，再赋值给data中定义的变量（或者计算属性中return prop中接收的数据，然后渲染这个计算属性）。经测试，虽然在vue工具devtools里面看，确实是已经有内容了，但是打印出来还是空，有点没明白。　

　　第一种方法可行，第二种方法未知。

vue项目打包后背景图片路径解析错误

发表于 2019-06-02 | 分类于 study

实例

　　给APP加了个默认背景，开发阶段运行正常，打包以后在手机上没有看到背景图片。

实现

问题

　　这样实现，在开发模式下是可以正常看到背景图片，但是打包以后就没有，所以我想是图片路径错误，在检查浏览器元素的时候发现图片定制变成了url(static/img/fan1.0d4dd90.jpg)。

原因

　　由上可以看出是自愿打包路径有误，打包后的资源使用了绝对根目录路径，因此将项目部署到特定目录下，其引入的资源路径无法被正确解析。

解决

　　1.在打包时需要使用相对路径来处理静态资源，更改build中资源发布路径配置（config/index.js,build对象）：将 assetsPublicPath: ‘/‘ 改为 assetsPublicPath: ‘./‘,重新打包后index.html可以正常访问，js和css资源也可以正常访问，但是css中引入的assets目录下的图片资源出错了，这也是上面的问题所在，所以还需要更改build文件夹下的util.js文件。（试验过成功）
　　
　　2.访问静态资源报错，也可以通过把图片放在static文件夹下，通过绝对路径访问。（没试验过）
　　通过上述方法改变对应配置文件以后，打包后的APP上还是没有显示背景图片，重点是打包后的dist文件直接在电脑浏览器访问是可以看到背景图的，所以我想应该是代码不兼容。后面一查，发现background-attachment:fixed;还真的是在移动端不兼容，background-attachment:fixed;是指明背景图的位置固定于视口，即随着页面的滚动背景也不动，相当于背景被设置在body上。于此选择了另一种替代方案，在body背景前添加了一个伪类，固定好位置不随滚动条滚动。问题这才算是真的解决了。

　　又发现一个问题，在页面跳转的时候，背景图加载缓慢，总是很明显的先看到白色背景再看到背景图，暂时还未找到解决办法。
